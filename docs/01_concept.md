# 01. コンセプト

## 課題

服選びで多くの人が最初に持つ感情は「今日何を着るか教えて」ではなく、
**「Instagramや街で見たあのコーデ、何が良いのか分からないけど好き」** である。

- 「好き」の理由 (シルエット/配色/抜け感/バランス) を言語化できない
- 憧れのスナップを保存しても、自分の服装には反映されない (Pinterest 止まり)
- 「あの感じ」を目指して服を買うと、手持ちと組めず無駄買いになる

## ソリューション

**スナップ翻訳アプリ** — 気になったコーデをカメラでパシャすると、1画面で以下を返す。

1. **魅力の分解** — テイスト / シルエット / 配色 (hexパレット) / 主役アイテム / 「何が良いのか」の言語化
2. **好みの蓄積** — 「ここが好き」をタップ保存 → 自分の好みプロファイルが育つ
3. **手持ちで再現** — クローゼットの服で組める近似コーデを最大3案 (使うアイテムのサムネイル付き)
4. **購買判断** — 優先度つき不足アイテム (使い回し見込み込み) + **買わなくていいもの**

```text
憧れのスナップ → 好み分析 → 手持ち服で再現 → 足りないものだけ買う
```

画像から似た服を探す「画像検索」でも、汎用の「コーデ提案」でもなく、
**"好きな見た目" を "実行できるコーデと購買判断" に変換する**ことが芯。

## ターゲットユーザー

服選びを時短したい・買い物の失敗を減らしたい一般ユーザー。
特にファッションアプリを使わない層 (男性含む) の「時短・失敗回避」ニーズ。

## 提供価値

- **服選び時短**: 憧れ→着られる形への変換を数十秒で
- **好みの可視化**: 単発の「いいね」で終わらせず、自分の好みデータベースになる
- **無駄買い防止**: 「買うべき1点」と「買わなくていいもの」を理由つきで返す

## コアUX原則

- **パシャ→即解析**: シャッターから結果画面まで一切ブロックしない。解析結果は1〜2秒からストリーミングで流れ込む
- **登録の摩擦最小**: クローゼット登録は「撮影1回 + 確認1タップ」。分類はAIが行う
- **断定しない**: パーソナルカラー (ブルベ/イエベ)・骨格タイプを断定せず、「近づけやすい」「相性が良い可能性」の見え方ベースで提案理由を必ず言語化する

## スコープ外 (やらないこと — MVP)

- アカウント / クラウド同期 / サーバDB (ローカルファースト — [ADR-0002](adr/0002-local-first-indexeddb.md))
- EC連携・購入導線
- 骨格診断・パーソナルカラー診断 (揉めやすく期待値を壊す)

## 成功指標

- 撮影→最初の解析テキスト表示まで 2秒以内 (体感の生命線)
- クローゼット登録の完了率 (登録が重いと死ぬカテゴリ)
- スナップ1枚あたりの「好き」タップ数 (好みDB化が回っているか)
