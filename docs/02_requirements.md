# 02. 要件定義

## 機能要件

### 必須 (MVP) — 実装済み

| # | 要件 | 実装 |
|---|---|---|
| F1 | カメラのライブビューファインダーで撮影できる (非対応環境はファイル選択にフォールバック) | `src/components/camera-capture.tsx` |
| F2 | 撮影画像をAIが分解する (テイスト/シルエット/配色/主役アイテム/魅力/再現の鍵) | `POST /api/analyze` |
| F3 | 解析結果は部分的に届き次第、逐次描画される (ストリーミング) | `src/lib/ai/client.ts` |
| F4 | 解析中に画面を離れても解析は完走し、結果が保存される | `src/lib/analysis-runner.ts` |
| F5 | 魅力の分解に「好き」をタップ保存でき、好みプロファイルとして集計される | `snap/[id]` + `snaps` 画面 |
| F6 | クローゼットに服を「撮影1回+確認1タップ」で登録できる (AI自動分類) | `POST /api/closet/categorize` + `closet` 画面 |
| F7 | 手持ち服で組める近似コーデ最大3案 + 優先度つき不足アイテム + 買わなくていいもの | `POST /api/match` |
| F8 | 全データはローカル (IndexedDB) に保存され、オフラインで閲覧できる | `src/lib/db/local.ts` |
| F9 | `AI_GATEWAY_API_KEY` 未設定でも全フローがモックで動く (課金ゼロのデモ/開発) | 各APIルートのフォールバック |
| F10 | ホーム画面追加できる PWA (manifest + apple-web-app メタ) | `src/app/manifest.ts` |

### あると良い (Post-MVP)

- 目的別コーデ (商談 / デート / 出社 / 旅行 / 登壇)
- 購入判断支援 (「この服を買ったら手持ちで何通り組めるか」)
- 好みプロファイルを解析プロンプトへフィードバック (パーソナライズ)
- クローゼットの背景除去・購入履歴連携
- Service Worker によるオフラインシェル

## 非機能要件

### パフォーマンス

| 項目 | 目標 | 現状 (実測) |
|---|---|---|
| 解析の初回表示 | シャッターから2秒以内に最初の解析テキストが見える | モックで初チャンク 399ms (実AI未測定) |
| 送信画像サイズ | 200KB級 (長辺1024px JPEG) にクライアント縮小 | 実装済み (`lib/image.ts`) |
| ブロッキング | シャッター後にユーザー操作を止める同期処理ゼロ | 実装済み (楽観的保存→即遷移) |
| タイムアウト | AIストリームは150sで打ち切り (サーバ maxDuration 120s + 余裕) | 実装済み |

### セキュリティ / プライバシー

- 画像・服データ・好みは端末内 (IndexedDB) 保存。外部送信はAI解析リクエストのみ
- APIボディは Zod で検証 (data URL 形式必須・2MB上限・不正は400)
- CSP / Permissions-Policy (`camera=(self)`) / HSTS を `next.config.ts` で静的定義
- 既知の残リスク: レート制限なし・照合テキスト長上限なし (ローカルファースト個人利用では許容、公開サービス化時に再評価)

### 可用性 / 運用

- サーバ状態なし (ステートレスAPI + クライアント保存) — サーバ障害時も保存済みデータは閲覧可
- AIキー未設定・AI障害時はモック/エラー表示に退避し、アプリ自体は動き続ける

## 制約条件

- AI呼び出しは Vercel AI Gateway 経由のみ (プロバイダ直結しない — [ADR-0004](adr/0004-two-model-strategy-ai-gateway.md))
- 表現ポリシー: ブルベ/イエベ・骨格の断定禁止、「似合う」と断言しない (`lib/ai/prompts.ts` が原本、テストで文言検証)
- カメラ (getUserMedia) は HTTPS 必須 → 実機検証は Vercel preview 以降

## 用語定義

| 用語 | 意味 |
|---|---|
| スナップ | ユーザーが撮影/取込した憧れコーデの画像とその解析結果 |
| 解析 (analyze) | スナップ1枚の魅力分解 (`SnapAnalysis`) |
| 照合 (match) | 解析結果×手持ち服 → 再現プラン (`MatchPlan`) |
| 好きポイント | 魅力分解のうちユーザーがタップ保存したラベル (`likedPoints`) |
| クローゼット | 手持ち服のローカルDB (`closetItems`) |
